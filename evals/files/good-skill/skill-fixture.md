# Good Skill

## Use This Skill When

- The user wants to improve an existing release-note skill.
- The current skill is too broad, too wordy, or unclear about when it should
  trigger.

## Do Not Use This Skill When

- The user wants a brand-new skill.
- The task is general repository maintenance outside the skill itself.

## Defaults

- Start with `SKILL.md`.
- Touch other files only when they would otherwise become stale.
- Prefer a few high-value edits over broad rewrites.

## Output

Give a brief summary of changes, or say plainly when no material change is
warranted.
